# NexaBank Fintech Analytics Dashboard

## Overview

This project analyzes a simulated fintech loan portfolio using **PostgreSQL**, **SQL**, **Power BI**, and **DAX**.

The objective is to transform raw banking data into actionable business insights by cleaning the data, building analytical queries, and creating an interactive executive dashboard for portfolio monitoring, customer risk analysis, and fraud analytics.

---

## Business Problem

Financial institutions need to continuously monitor:

- Loan portfolio performance
- Customer default risk
- Transaction activity
- Fraud indicators

The goal of this project is to provide decision-makers with an interactive dashboard that supports data-driven lending and operational decisions.

---

## Dataset

The project uses a simulated fintech banking database containing:

- Customers
- Loans
- Transactions
- Fraud Flags
- Accounts
- Cards
- Merchants
- Marketing Interactions
- Support Tickets

## Power BI Dashboard

The dashboard consists of four pages.

### 1. Executive Overview

- Portfolio KPIs
- Loan status distribution
- Loans by product
- Default rate by loan size
- Default rate by income band

---

### 2. Customer Risk Analysis

- Default rate by age group
- Default rate by income band
- Top borrowers
- Customer segment analysis
- Interactive slicers

---

### 3. Operations & Fraud Analytics

- Transaction KPIs
- Fraud investigations
- Fraud reasons
- Payment channels
- Transaction status

---

### 4. Executive Summary

Summarizes:

- Business objectives
- Key findings
- Business recommendations
- Potential risks
- Future improvements

---

## Key Business Insights

- Active loans represent **69.8%** of the portfolio.
- Portfolio value exceeds **€107 million**.
- Personal Loans are the largest product category.
- Loans above **€20k** have the highest default rate.
- Customers earning under **€30k** present the highest default risk.
- Mobile App is the dominant payment channel.
- Most fraud investigations have been successfully resolved.
